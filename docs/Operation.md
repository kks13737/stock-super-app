# Operation: 나만의 주식관리 슈퍼앱

## 1. 실행 방법

### 1.1 백엔드
- `backend` 폴더로 이동한다.
- `uv sync`로 의존성을 설치한다.
- `uv run uvicorn app.main:app --host 127.0.0.1 --port 8000`으로 서버를 실행한다.
- 브라우저에서 `http://127.0.0.1:8000`에 접속한다.

예시 명령:
```bash
cd backend
uv sync
uv run uvicorn app.main:app --host 127.0.0.1 --port 8000
```

### 1.2 프론트엔드
- 별도 프론트 서버는 필요하지 않다.
- 백엔드가 `frontend/index.html`, `frontend/styles.css`, `frontend/app.js`를 함께 서빙한다.
- 루트 페이지에서 화면을 확인한다.

예시 명령:
```bash
start http://127.0.0.1:8000
```

## 2. 사용 방법

### 2.1 매매일지 입력
1. 캘린더에서 날짜를 선택한다.
2. 종목명, 매수/매도 구분, 수량, 가격, 메모를 입력한다.
3. 저장 후 해당 날짜에서 기록을 확인한다.

### 2.2 뉴스 수집
1. 자동 수집은 매일 9시(KST)에 실행된다.
2. 수동 수집 버튼을 누르면 즉시 네이버 증권 뉴스를 다시 수집한다.
3. 수집 결과는 뉴스 목록 화면에서 확인한다.

### 2.3 공포탐욕지수 수집
1. 자동 수집은 6시간마다 실행된다.
2. 수동 수집 버튼으로 즉시 최신 지수를 가져온다.
3. 최신 지수와 수집 시각을 대시보드에서 확인한다.

## 3. 자주 발생하는 에러

### 3.1 네트워크 오류
- 증상: 수집 실패, 타임아웃, 연결 거부
- 원인: 대상 사이트 일시 장애, 네트워크 불안정
- 대응: 재시도 후에도 실패하면 로그를 확인한다.

### 3.2 HTML 구조 변경
- 증상: 특정 항목이 비어 있거나 파싱 실패
- 원인: 네이버 또는 CNN 페이지 구조 변경
- 대응: `crawlers` 모듈의 선택자와 파싱 규칙을 수정한다.

### 3.3 중복 저장
- 증상: 동일 기사나 지수가 여러 번 저장됨
- 원인: 수동 수집과 자동 수집이 겹침
- 대응: URL 또는 소스 기반 고유 키로 중복 체크를 적용한다.

### 3.4 시간대 오류
- 증상: 9시 수집이 한국 시간 기준과 다르게 실행됨
- 원인: 서버 시간대가 UTC로 설정됨
- 대응: 모든 스케줄 기준을 KST로 고정한다.

### 3.5 `uv` 의존성 다운로드 실패
- 증상: `uv sync` 또는 `uv run`이 PyPI에 접근하지 못함
- 원인: 네트워크 차단 또는 프록시 정책
- 대응: 네트워크 허용 후 다시 실행하거나, 기존 `.venv`가 있으면 재사용한다.

## 4. 운영 체크리스트
- 스케줄러가 활성화되어 있는지 확인
- 수집 로그가 남는지 확인
- 중복 저장 방지가 정상 동작하는지 확인
- HTML 파싱 실패 알림이 있는지 확인
- KST 기준 시간이 맞는지 확인

## 5. 권장 운영 정책
- 수집 작업은 실패 시 최소 1회 재시도한다.
- 파서 실패 시 원본 HTML 일부를 로그에 남긴다.
- 스케줄러와 수동 수집은 동일 서비스 계층을 사용해 동작을 일관화한다.
- 운영 중에는 수집 성공률과 최근 실패 원인을 주기적으로 확인한다.
