# PRD: 나만의 주식관리 슈퍼앱

## 1. 제품 개요
나만의 주식관리 슈퍼앱은 주식 매매 기록, 투자 관련 뉴스 수집, 공포탐욕지수 추적을 한 곳에서 관리하는 개인용 투자 보조 도구다.  
핵심 목표는 사용자가 투자 판단에 필요한 정보와 자신의 매매 히스토리를 빠르게 확인하도록 돕는 것이다.

## 2. 목표
- 주식 매매일지를 날짜 기준으로 쉽게 기록한다.
- 네이버 증권 뉴스와 CNN 공포탐욕지수를 자동 수집한다.
- 수동 수집도 지원해 즉시 최신 데이터를 반영한다.
- 수집된 데이터를 한 화면에서 조회 가능하게 한다.

## 3. 주요 사용자
- 개인 투자자
- 데일리 트레이딩을 기록하는 사용자
- 뉴스와 심리 지표를 함께 보고 싶은 사용자

## 4. 핵심 기능

### 4.1 주식 매매일지 기능
- 특정 주식 종목의 매매 기록을 남길 수 있는 캘린더 제공
- 날짜별로 매수, 매도, 보유 메모를 기록
- 종목명, 매매 구분, 수량, 가격, 메모를 저장
- 날짜를 눌러 해당 일자의 거래 기록을 확인

### 4.2 뉴스 기사 수집
- 매일 9시(KST)에 전날 데이터를 자동 수집
- 수동 클릭으로 즉시 수집 가능
- 수집 대상: 네이버 증권 뉴스
  - 대상 URL: `https://finance.naver.com/news/news_list.naver?mode=LSS2D&section_id=101&section_id2=258`
- Python BS4 기반 크롤러 사용
- 수집 항목 예시
  - 기사 제목
  - 기사 링크
  - 언론사
  - 작성일
  - 요약 또는 본문 일부

### 4.3 공포탐욕지수 수집
- 6시간마다 자동 수집
- 수동 클릭으로 즉시 수집 가능
- 수집 대상: CNN Fear and Greed Index
  - 대상 URL: `https://edition.cnn.com/markets/fear-and-greed`
- 수집 항목 예시
  - 현재 지수 값
  - 감정 구간
  - 수집 시각

## 5. 사용자 시나리오
1. 사용자가 날짜를 선택하고 특정 종목의 매매 내역을 입력한다.
2. 사용자는 오늘 수집된 뉴스와 공포탐욕지수를 확인한다.
3. 사용자는 필요 시 수동 수집 버튼을 눌러 최신 데이터를 다시 가져온다.
4. 사용자는 과거 날짜를 눌러 당시의 매매 기록을 검토한다.

## 6. 기능 요구사항
- 사용자는 캘린더에서 날짜별 매매 기록을 저장하고 조회할 수 있어야 한다.
- 시스템은 매일 9시(KST)에 네이버 증권 뉴스를 자동 수집해야 한다.
- 시스템은 6시간마다 공포탐욕지수를 자동 수집해야 한다.
- 사용자는 수동 버튼으로 언제든지 수집을 실행할 수 있어야 한다.
- 수집 결과는 중복 저장을 방지해야 한다.
- 수집 실패 시 실패 이력과 원인을 확인할 수 있어야 한다.

## 7. 비기능 요구사항
- 수집 작업은 백그라운드에서 실행되어 UI 응답을 방해하지 않아야 한다.
- 크롤링 대상 구조 변경에 대비해 파서 로직을 분리해야 한다.
- 저장 데이터는 날짜 기반 조회가 빠르도록 설계해야 한다.
- 운영 중 에러를 추적할 수 있도록 로그를 남겨야 한다.

## 8. 제외 범위
- 자동 매매
- 증권사 계좌 연동
- 실시간 시세 고빈도 처리
- 복수 사용자 계정 및 권한 관리
